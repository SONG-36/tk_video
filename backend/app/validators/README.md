# validators

## 模块职责

描述未来统一校验状态闸门、Schema、跨对象引用、目标型号切换失效和路径归属的职责。

## 输入

当前状态、目标状态、业务对象、Schema 校验结果、系列/型号/批次引用和文件路径。

## 输出

通过或阻断结果、错误位置、原因、受影响对象和需要失效的后续步骤。

## 依赖规则

按被校验对象读取相应 `rules/*.json`；状态和质量硬规则直接服从 `00_design.md` 第 8、9、17、18、20、21 章，不新增规则投影。

## 依赖 Schema

`backend/schemas/` 下全部当前 Schema。

## 前置状态 / 通过条件

本模块不推进状态；未来每个业务模块在接受输入和产出结果时调用对应校验，前置闸门不满足即阻断。

## ID 绑定要求

未来校验 `selected_variant_id` 数组引用、型号唯一性、三类 ID 跨文件一致、视频/镜头批次归属、任务包上下文、输出路径和目标型号切换后的失效范围。

还需校验：产品首次清晰出现不晚于 8 秒、产品证明不晚于 12 秒开始、每条正式视频 hook/proof/CTA 完整、剧情不能压过产品，以及不允许直接复制来源视频、台词、画面、人物、品牌或音乐。时间轴算法和剧情平衡判定尚未定义时只能报告 unresolved，不能硬编码临时阈值以外的解释。

## 禁止行为

不得自动修正规则或数据、绕过人工确认、替模块推进状态、把历史 `product_id` 当主结构或假装 README 已实现校验。

## 后续实现边界

当前只定义职责，不创建源码。B/C 与第 21 章、三档画像与布尔字段均为设计未裁决约束；validators 在人工裁决前只能报告，不能选择解释。
