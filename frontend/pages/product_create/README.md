# 新建商品页

本文档描述前端页面职责，不是业务规则权威源；如与根目录 `00_design.md` 冲突，以 `00_design.md` 为准。

## 页面目标

建立商品系列基础信息。

## 前置状态

`draft`。

## 进入条件

用户发起新建商品。

## 输入 / 展示字段

必填：产品名称、目标平台、目标市场、当前业务阶段。选填：商品图片、1688/TikTok/Amazon 链接、供应商资料、价格区间、采购数量、备注。业务阶段选项为选品验证、已采购样品、准备拍视频、准备上架、已上架待优化。

## 用户动作

填写、修改并提交商品基础信息。

## 输出对象

带 `product_series_id` 的商品基础信息 JSON。

## 依赖后端模块

未来由 `validators` 校验必填项；当前后端模块清单没有独立商品创建模块，不在本任务中新增。

## 依赖 Schema

第 18 章定义基础信息对象，但当前没有独立 `basic_info` Schema；`product_series.schema.json` 只作系列标识导航，不能假装已覆盖草稿阶段。

## 通过条件

产品名称非空，目标平台、目标市场和当前业务阶段已选择。

## 失败 / 阻断表现

任一必填项缺失时保留 `draft`，显示缺失字段，不能进入类目确认。

## 型号和 ID 约束

此时只建立 `product_series_id`，不提前创建目标型号或 `selected_variant_id`。

## 下一状态

`basic_info_completed`。

## 未裁决约束

基础信息缺少独立 Schema，且当前系列 Schema 要求后续型号字段；实现前需人工补齐契约。
