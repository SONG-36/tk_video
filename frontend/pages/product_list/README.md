# 商品列表页

本文档描述前端页面职责，不是业务规则权威源；如与根目录 `00_design.md` 冲突，以 `00_design.md` 为准。

## 页面目标

展示所有已创建商品及每个商品当前卡住的步骤。

## 前置状态

无固定前置状态；可展示从 `draft` 到 `completed` 或 `blocked` 的商品。

## 进入条件

进入商品库即可查看；具体操作受所选商品当前状态限制。

## 输入 / 展示字段

商品名称、目标平台、目标市场、平台类目、商品分类、当前状态、缺失材料数量、最近更新时间、操作按钮。

## 用户动作

继续处理、查看报告、查看任务包、查看输出结果、删除商品。

## 输出对象

无独立业务数据对象；输出所选 `product_series_id` 和用户动作意图。

## 依赖后端模块

`validators` 用于未来判断操作是否符合状态闸门；报告、任务包和输出分别读取对应既有结果。当前没有专用商品列表模块。

## 依赖 Schema

`product_series.schema.json`；查看报告、任务包时分别参考 `judgement_report.schema.json`、`codex_task_package.schema.json`。

## 通过条件

第 7 章未定义独立通过条件；页面只能把用户路由到当前状态允许的目标页。

## 失败 / 阻断表现

数据缺失或操作不满足状态时显示当前状态和阻断原因，不自动推进。

## 型号和 ID 约束

列表以 `product_series_id` 定位商品；型号级结果必须显示其 `variant_id`，不能混入其他型号。

## 下一状态

本页不改变状态；“继续处理”进入当前状态对应页面。
