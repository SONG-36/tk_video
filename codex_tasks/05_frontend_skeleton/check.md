# 任务 05 人工验收

## 人工验收清单

- [ ] 执行前完整读取了 `00_design.md`。
- [ ] 15 个页面 README 均包含目标、字段、动作、输出、通过条件和状态依赖。
- [ ] 页面严格按第 4、7 章顺序排列。
- [ ] 前端职责明确为流程控制和展示，不是聊天框。

## 文件是否存在

- [ ] 商品、类目、型号和画像相关 7 个页面 README 存在。
- [ ] 报告、材料、图片、创意 4 个页面 README 存在。
- [ ] 批次、镜头、任务包、输出 4 个页面 README 存在。
- [ ] `frontend/README.md` 和 `frontend/pages/README.md` 存在。
- [ ] 页面目录中没有 UI 源码、路由或构建配置。

## 内容是否符合 00_design.md

- [ ] 页面字段、用户动作和通过条件可回溯到第 7 章。
- [ ] 前置状态和下一状态与第 8、9 章一致。
- [ ] 类目确认明确需要人工操作。
- [ ] 目标型号选择之后的页面均绑定当前型号。
- [ ] 切换 `selected_variant_id` 的失效和复验规则完整。
- [ ] output_viewer 只读取批次规划中存在的视频目录。

## 是否存在越权修改

- [ ] `git diff` 只包含允许的前端 README 和必要问题记录。
- [ ] 未修改 docs 正文、rules、backend、Schema、脚本、任务或 `00_design.md`。
- [ ] 未创建 UI、API 客户端、数据库、登录或外部模型调用。

## 是否存在业务规则被改写

- [ ] 没有调序、合并、跳过或新增页面。
- [ ] 没有新增自动确认、字段默认值、状态或通过条件。
- [ ] 没有把页面 README、docs、rules 或 Schema 当权威源。
- [ ] 冲突只记录，没有通过交互设计擅自解决。

## 是否可以进入下一任务的判断标准

只有 15 个页面说明完整、状态与字段可追溯、型号切换约束明确、允许范围外零改动且没有 UI 实现时，才可以进入任务 06。
