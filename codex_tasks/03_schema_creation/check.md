# 任务 03 人工验收

## 人工验收清单

- [ ] 执行前已完整读取 `00_design.md`。
- [ ] 10 个 Schema 均可作为 JSON 解析，并声明统一 draft。
- [ ] 每个 Schema 说明对应对象、必填字段、归属和绑定。
- [ ] 无法由 Schema 表达的约束被明确交给后续 validators，而非遗漏。

## 文件是否存在

- [ ] 系列、型号、画像、报告、材料、图片、创意七个对象 Schema 存在。
- [ ] 视频批次、镜头计划、Codex 任务包三个三类 ID 对象 Schema 存在。
- [ ] `backend/schemas/README.md` 明确 Schema 不是权威源。

## 内容是否符合 00_design.md

- [ ] `selected_variant_id` 必须对应当前系列的某个 `variant_id`，关系有明确说明。
- [ ] 批次、镜头和任务包都要求 `product_series_id + variant_id + batch_id`。
- [ ] 视频数量为 3—5，Revision 默认可为空，单型号仍使用型号对象。
- [ ] 图片、创意、材料和报告均有目标型号归属。
- [ ] 历史单型号兼容对象未成为当前主 Schema。

## 是否存在越权修改

- [ ] `git diff` 只包含允许的 Schema 文件和必要冲突记录。
- [ ] 未修改规则、业务模块、页面、脚本、测试、任务或 `00_design.md`。
- [ ] 未实现 validator、数据库、API、登录或外部模型集成。

## 是否存在业务规则被改写

- [ ] Schema 没有改变字段含义、枚举、数量或通过条件。
- [ ] 没有通过宽松 Schema 取消型号或三类 ID 绑定。
- [ ] 没有把 Schema、docs 或 rules 宣称为权威源。
- [ ] 设计缺口已记录，未用自创字段静默填补。

## 是否可以进入下一任务的判断标准

只有 10 个 Schema 语法有效、绑定完整、必填字段可追溯、范围外零改动，且所有无法表达的关系有后续校验说明时，才可以进入任务 04。
