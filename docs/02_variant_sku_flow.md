# 商品系列、型号与 SKU 流程

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。
>
> 来源章节：第 5、6 章。

## 对象关系

```text
Product Series 商品系列
→ Variant / SKU 型号
→ Video Batch 视频批次
→ Shot Plan 镜头任务包
```

- 商品系列：业务商品集合，保存平台类目、目标市场和真正共享的定位。
- 型号 / SKU：可单独选择、拍摄和销售，保存外观、尺寸、参数、颜色、配件、图片和卖点差异。
- 视频批次：绑定一个目标型号的一组 3—5 条视频测试方案。
- 镜头任务包：批次内 A/B 级视频的镜头级执行资产。

一个批次只能绑定一个本次拍摄型号。不同型号的图片、参数、配件和卖点默认不能混入同一批次。

## 型号差异判断

型号录入后至少比较：

1. 型号名称和 SKU 编码。
2. 外观结构与颜色。
3. 尺寸和重量。
4. 功能与参数。
5. 按钮、接口和显示区域。
6. 包装和配件。
7. 价格与主推卖点。
8. 产品图片是否可以共用。
9. 适合的视频分型是否相同。
10. 合规与安全信息是否相同。

差异会影响画面、参数、卖点、配件或合规表达时，必须分型号生产内容，不能随机混用。

## 目标型号选择

进入型号级画像前必须选择 `selected_variant_id`。判断报告、材料检查、图片资产、创意方向、视频批次、镜头资产和任务包都必须携带该标识。商品只有一个型号时，也创建默认型号并自动选中，避免形成两套数据逻辑。

## 产品版本 / Revision 预留

Revision 不是默认必经流程，只在以下情况启用：

- 新旧款结构不同。
- 新旧款参数不同。
- 新旧款图片不能混用。
- 需要制作新旧款对比视频。
- 需要判断旧素材是否仍可使用。

默认不创建版本对象、不增加版本确认页面、不把版本判断加入状态机主路径，`optional_revision` 默认是 `null`。
