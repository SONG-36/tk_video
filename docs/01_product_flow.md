# 产品完整流程

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源。来源：第 4、9 章；冲突时以 `00_design.md` 为准。

## 完整总流程

商品输入 → 平台类目确认 → 型号 / SKU 录入 → 型号差异判断 → 选择本次拍摄型号 → 型号级商品画像补全 → 型号级视频分型判断 → 判断报告确认 → 材料检查 → 型号级产品图片资产检查 → 创意方向设定 → 视频批次规划 → 镜头资产规划 → Codex 任务包导出 → VSCode / Codex 生成批次内容资产 → 前端读取输出结果。

商品分类是型号级判断报告的必要输入，不单独占用主流程页面。

## 页面进入条件

- 基础信息完成后才能确认类目；类目人工确认后才能确认型号业务信息。
- 型号录入后才能比较差异；差异判断后才能选择目标型号。
- 选择目标型号后才能补全画像；画像完成后才能生成分型和报告。
- 报告确认后才能检查材料；材料通过后才能检查目标型号图片。
- 图片检查后才能设定创意；创意确认后才能规划批次。
- 批次规划后才能规划镜头；镜头规划后才能导出任务包。
- 任务包导出后才能进入 Codex 输出结果页。

## 主流程说明

主流程最小闭环是确认一个可拍摄目标型号，为它准备材料和图片，规划一批视频并输出镜头级任务包。切换 `selected_variant_id` 后必须重新验证所有型号级步骤；只有明确标为系列共享的材料可沿用。
