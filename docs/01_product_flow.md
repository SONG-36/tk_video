# 产品完整流程

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。
>
> 来源章节：第 4、9 章。

## 完整总流程

```text
商品输入
→ 平台类目确认
→ 型号 / SKU 录入
→ 型号差异判断
→ 选择本次拍摄型号
→ 型号级商品画像补全
→ 型号级视频分型判断
→ 判断报告确认
→ 材料检查
→ 型号级产品图片资产检查
→ 创意方向设定
→ 视频批次规划
→ 镜头资产规划
→ Codex 任务包导出
→ VSCode / Codex 生成批次视频内容资产
→ 前端读取输出结果
```

商品分类是判断报告的必要规则输入，但在型号级商品画像分析中执行，不增加独立主流程页面。主流程闭环是：确认一个可拍摄目标型号，为该型号准备材料和图片，规划一批视频并输出镜头级任务包。

## 页面进入条件

1. 商品系列基础信息未完成：不能进入平台类目确认。
2. 平台类目未确认：不能录入并确认型号业务信息。
3. 型号 / SKU 未录入：不能进行型号差异判断。
4. 型号差异未判断：不能选择本次拍摄型号。
5. 本次拍摄型号未选择：不能补全型号级商品画像。
6. 型号级商品画像未完成：不能生成视频分型和判断报告。
7. 判断报告未确认：不能进入材料检查。
8. 材料检查未通过：不能进入型号级产品图片资产检查。
9. 目标型号图片资产未检查：不能进入创意方向设定。
10. 创意方向未确认：不能进入视频批次规划。
11. 视频批次未规划：不能进入镜头资产规划。
12. 镜头资产未规划：不能导出 Codex 任务包。
13. 任务包未导出：不能进入 Codex 输出结果页。

## 目标型号切换

切换 `selected_variant_id` 后必须重新验证所有型号级步骤。只有明确标记为系列共享的材料可以继续沿用；其他型号的参数、图片、配件、卖点和限制不得进入当前流程。
