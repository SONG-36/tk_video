# 项目级开发计划

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。
>
> 来源章节：第 1—24 章；阶段顺序用于落实完整设计，不改变业务范围。

## 阶段 0：项目结构与文档拆分

建立完整目录、职责说明、阅读版、任务与校验入口，记录设计问题。

## 阶段 1：规则 JSON 与 Schema 建立

逐章完整提取规则，完成 TikTok 创意结构规则、爆款素材库规则、创意迁移规则、`viral_pattern_card` / `product_adapted_pattern` 数据结构预留、对象 Schema、跨对象引用及三类 ID 绑定校验。

## 阶段 2：后端模块骨架

定义判断器、检查器、规划器、生成器、读取器接口及模块测试，不改变业务含义。

## 阶段 3：前端页面骨架

按既定顺序建立全部页面、字段、状态依赖和流程闸门，不跳页、不合并业务步骤。

## 阶段 4：流程状态机与闸门校验

实现完整状态枚举、进入条件、阻断原因、目标型号切换失效和共享材料复验。

## 阶段 5：任务包导出

聚合已确认数据，校验 `product_series_id + variant_id + batch_id`，生成 Markdown/JSON 任务包。

## 阶段 6：Codex 输出读取

按批次规划读取输出文件、校验结构并支持前端查看和质检状态。

## 阶段 7：样例商品跑通

完整跑通车载吸尘器与无线直发梳案例，覆盖型号隔离、材料降级、真人依赖和不同分型。

## 阶段 8：前端体验优化

在不改变流程和规则的前提下优化表单、状态提示、资产标注、批次编辑和输出浏览。

## 阶段 9：后续能力接入

先预留 `knowledge_base/viral_patterns/`、`adapted_patterns/`、`rejected_patterns/` 的目录契约；后续再实现本地素材库管理。在核心流程稳定且接口隔离后，才评估图片识别 / VLM / 视频模型 API，以及素材抓取和自动分析；抓取、平台自动分析和真实素材库均不进入当前实现。外部能力不能绕过人工类目确认、型号绑定、规则判断和流程闸门。

## 第 17/18 章同步里程碑

- TikTok 创意结构规则同步。
- 爆款素材库与创意迁移规则同步。
- 知识库目录仅预留，不创建真实抓取或分析能力。
- Schema、页面契约、任务包和输出读取先承载新增字段；validators、导出器、读取器和页面实现留待对应代码任务。

## 代码阶段技术栈确认

以下内容是进入 `codex_tasks_code/08_frontend_project_init` 前的人工工程选型记录，只约束代码施工方式，不新增或修改 `00_design.md` 的业务规则：

| 项目 | 人工确认结果 | 任务 08 边界 |
|---|---|---|
| 前端框架 | React | 只创建最小空应用壳 |
| 语言 | TypeScript | 不定义业务类型或字段 |
| 构建工具 | Vite | 只提供最小 dev/build 配置 |
| 包管理器 | npm | 任务 08 才允许安装已批准依赖并生成唯一 npm 锁文件 |
| 测试工具 | Vitest | 只验证最小应用入口可挂载 |
| 样式方案 | 普通 CSS | 不引入 Tailwind 或 UI 组件库 |
| 路由 | 任务 09 再引入 React Router | 任务 08 不引入路由库或路由实现 |
| 状态管理 | 任务 10 再实现本地状态机 | 任务 08 不引入 Redux、Zustand 或其他状态库 |
| 后端 | 任务 08 不初始化 | 不创建后端服务或后端代码 |
| 数据 | 任务 08 不实现真实读写 | 不读取或写入 `data/`、`outputs/` |
| API | 任务 08 不创建 | 不创建 API client 或接口调用 |

任务 08 的批准组合为：`React + TypeScript + Vite + npm + Vitest + 普通 CSS`。本确认不执行初始化、不安装依赖、不创建 `package.json`、锁文件、配置或代码；实际初始化只能在单独执行任务 08 时进行。

## 阶段执行原则

- 各阶段只决定工程施工顺序，不删减 `00_design.md` 中的任何页面、规则、对象或输出。
- 每阶段必须通过对应 `codex_tasks/` 验收后才能继续。
- 发现设计冲突时只记录到 `docs/99_design_issues.md`，不得在开发计划中自行裁决。
