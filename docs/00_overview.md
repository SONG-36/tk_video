# 系统总览

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。
>
> 来源章节：第 1、2、3、5、24 章。

## 文档定位

`00_design.md` 单独定义产品定位、业务对象、主流程、页面、规则、数据结构、输出规范和验收边界，是唯一权威源。`docs/` 和 `codex_tasks/` 分别是阅读版与执行指令。

## 项目定位

本系统是面向 TikTok 电商商品短视频生产的流程式内容决策工作台。它先确认具体型号适合拍什么、缺什么以及禁止编造什么，再把标准任务交给 VSCode / Codex。前端控制流程并展示结果，后端执行规则判断和文件生成，Codex 生成可追踪到目标型号的内容资产。

## 核心原则

- 唯一权威源：变更先进入 `00_design.md`，再同步从属内容。
- 流程闸门：每一步有通过条件，前一步未完成不能继续。
- 职责边界：前端不是聊天框；后端不是大模型服务；执行器不能绕过商品判断、SKU 约束和闸门。
- 型号隔离：一个批次只绑定一个目标型号，不混用图片、参数、配件、卖点和限制。

## 复杂度边界

默认对象链为 `Product Series → Variant / SKU → Video Batch → Shot Plan`。不把组织、供应商、渠道、仓库、复杂版本树或素材知识图谱加入主流程。Revision 仅在同型号新旧款确有结构、参数、图片或素材兼容差异时启用，默认不创建对象、不新增页面、不进入状态机。

## 总结

最终核心资产是“型号级商品判断报告 + 目标型号视频批次规划 + 镜头级视频生产任务包”。系统重点防止型号混用、材料不足、未判断分型、创意无约束和任务包脱离业务规则。
