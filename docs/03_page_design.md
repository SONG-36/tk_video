# 页面设计

> 本文档来源于 `00_design.md`，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。
>
> 来源章节：第 7、17、18 章。

## 页面顺序

商品列表 → 新建商品 → 类目确认 → 型号 / SKU 录入 → 型号差异判断 → 本次拍摄型号选择 → 型号级商品画像补全 → 型号级判断报告 → 材料检查 → 型号级产品图片资产 → 创意方向设定 → 型号级视频批次规划 → 镜头资产规划 → Codex 任务包导出 → 输出结果查看。

## 1. 商品列表页

- 目标：展示所有已创建商品以及当前卡住的步骤。
- 字段：商品名称、目标平台、目标市场、平台类目、商品分类、当前状态、缺失材料数量、最近更新时间、操作按钮。
- 动作：继续处理、查看报告、查看任务包、查看输出结果、删除商品。
- 输出与通过条件：第 7 章未定义独立业务输出或通过条件；页面依据当前状态路由到允许的下一步。

## 2. 新建商品页

- 目标：建立商品对象。
- 必填：产品名称、目标平台、目标市场、当前业务阶段。
- 选填：商品图片、1688 链接、TikTok 竞品链接、Amazon 竞品链接、供应商资料、价格区间、采购数量、备注。
- 业务阶段：选品验证、已采购样品、准备拍视频、准备上架、已上架待优化。
- 通过条件：四个必填字段均已选择或填写。
- 输出：商品基础信息 JSON。

## 3. 类目确认页

- 目标：识别并人工确认平台类目，不能让 AI 猜测后自动继续。
- 展示：产品名称、候选一级类目、候选二级类目、置信度、推荐理由、风险提示。
- 动作：确认推荐类目、手动修改类目、返回修改商品名称。
- 通过条件：`platform_category` 已确认；必要时 `sub_category` 已确认；用户点击确认。
- 输出：类目确认 JSON。

## 4. 型号 / SKU 录入页

- 目标：记录系列下所有可独立拍摄和销售的型号。
- 字段：型号名称、卖家 SKU、平台 SKU、颜色、尺寸、核心参数、配件、价格、型号图片。
- 通过条件：至少存在一个型号。
- 输出：型号列表；单型号商品也创建默认型号。

## 5. 型号差异判断页

- 目标：判断哪些字段可共享、哪些必须隔离、是否需要分别制作视频。
- 比较：外观、参数、功能、配件、图片、卖点、合规信息。
- 输出：共享字段、不可共享字段及分型号生产结论。
- 通过条件：型号差异已经判断。

## 6. 本次拍摄型号选择页

- 目标：选择本批次唯一目标型号。
- 展示：商品系列、型号、SKU 编码和关键差异。
- 动作：确认 `selected_variant_id`。
- 通过条件：目标型号已选择。
- 约束：后续页面持续显示系列、目标型号和 SKU；切换型号会使未确认画像、报告、图片绑定、批次和镜头规划失效并重新检查。

## 7. 型号级商品画像补全页

- 目标：用结构化选择补齐影响判断与生产的型号特征，不让运营自由乱写。
- 基础字段：体积、真人出镜需求、大场景需求、功能明确程度、审美/款式/效果依赖、是否适合对比、包装展示价值。
- 风险字段：电子/电器、电池、接触人体、儿童、宠物、食品、高单价、认证或合规要求。
- 卖点类型：功能效果、便携、外观颜值、价格优势、材质品质、使用场景、痛点解决、情绪价值、安全性、省时间、省空间。
- 冲突检查：识别画像与类目明显冲突；严重冲突不能继续，轻微冲突必须由用户确认。
- 通过条件：基础和风险字段完整；至少一个卖点；无严重冲突；轻微冲突已确认。
- 输出：型号级商品画像 JSON。

## 8. 型号级判断报告页

- 目标：整合类目、商品分类、内容难度、视频分型、类目打法、风险和缺失信息，作为后续内容决策底稿。
- 结构：顶部结论卡、内容难度评分、视频分型推荐、推荐拍法排序、类目打法建议、风险提醒、缺失信息、下一步动作。
- 通过条件：报告已生成；用户已阅读风险；用户确认报告；严重缺失项进入材料检查处理。
- 输出：绑定目标型号的判断报告。

## 9. 材料检查页

- 目标：在脚本、分镜和提示词生成前检查材料是否够用。
- 分类：商品基础、内容生产、创意表达、产品图片、视频素材、合规与限制、竞品参考。
- 输出：必须、建议、风险和缺失项以及能否继续。
- 通过条件：服从第 12 章材料规则；必须项齐全，建议缺失已确认，风险缺失已写入禁止编造。

## 10. 型号级产品图片资产页

- 目标：判断原始图片是否支撑 AI 视频，并标记镜头可调用资产。
- 类型：`product_reference`、`product_detail`、`usage_scene`、`before_after`、`packaging`、`style_reference`、`motion_reference`。
- 动作：上传、标注类型和用途、查看缺图及能力降级。
- 通过条件：图片已上传或明确缺失；完成类型与用途标记；生成完整度、缺图和能力限制。
- 输出：绑定目标型号的图片资产清单。

## 11. 创意方向设定页

- 目标：把普通创意语言翻译为运镜、质感、节奏、光线、参考素材和提示词要求。
- 字段：创意风格、剧情结构、表达强度、真人限制、重点卖点、自由创意输入；并支持 `tiktok_creative_structure`、hook 偏好、TikTok 原生风格要求、是否使用爆款模式、`viral_pattern_card` 选择和 `product_adapted_pattern` 结果。
- 提醒：展示跨文化适配和产品适配风险，区分可迁移结构与不可复制内容；这些能力属于页内输入与约束，不新增主流程页面。
- 通过条件：至少一个风格和剧情结构；已选强度与真人限制；确认创意优先级；系统完成专业翻译；用户确认。
- 输出：目标型号的创意方向和系统专业翻译。

## 12. 型号级视频批次规划页

- 目标：围绕一个目标型号规划 3—5 条用于测试的差异化短视频。
- 批次目标：测试不同分型、开头钩子、卖点、场景、创意风格、用户痛点、爆款模式、情绪触发、前 3 秒机制或本地化迁移方式。
- 每条字段：原有字段外，增加或预留 `hook_type`、`first_3_seconds_text`、`first_3_seconds_visual`、`scroll_stop_reason`、`emotional_trigger`、`script_structure`、`retention_design`、`key_message`、`proof_point`、`cta`、`tiktok_native_style`、`text_overlay_plan`、`sound_or_voiceover_plan`、`viral_pattern_ref`、`adapted_pattern_ref`。
- 批次约束：同一批次内不应让 3 条视频使用同一种前 3 秒机制。
- 通过条件：批次 ID 和数量已确认；逐视频必填项完整；图片支撑等级；不满足条件者不能进入 A 级；用户确认批次。
- 输出：绑定三类 ID 的 `video_batch_plan`。

## 13. 镜头资产规划页

- 目标：把批次视频拆成可执行镜头资产。
- 资产：每条视频一个 Creative Board、一个 Storyboard；每个镜头一个 Shot Card 和一个 Seedance Prompt；使用爆款模式时，把 `viral_pattern_card` / `product_adapted_pattern` 转成镜头而非只保留概念。
- 逐镜头补充：情绪触发、反转推进、产品证明、剧情铺垫角色、产品首次出现时间、产品证明开始时间、剧情/产品平衡、证明场景、CTA 画面、字幕和声音计划。
- 通过条件：A/B/C 视频分别达到第 16 章要求；镜头绑定图片或标记不需要；标记失真风险；明确服务的卖点或剧情目标。
- 输出：批次级 `shot_asset_plan`。

## 14. Codex 任务包导出页

- 目标：将前序结果整理为标准任务包；前端不直接调用 Codex 插件，只生成文件。
- 输出：商品、报告、图片、创意、批次、镜头计划、TikTok 创意结构、爆款模式迁移信息及 `{product_series_id}_{variant_id}_{batch_id}_codex_task.{md,json}`。
- 通过条件：镜头资产已规划，任务包三类 ID 与目标型号和批次一致。

## 15. 输出结果查看页

- 目标：读取 Codex 输出并展示脚本、分镜、提示词和拍摄清单。
- 读取：`outputs/{product_series_id}/{batch_id}/batch_summary.md` 和批次规划中实际存在的视频目录。
- 展示：创意画板、脚本、故事板、镜头卡、镜头提示词、图片输入计划、剪辑计划、质检、缺失素材，以及 hook、proof、CTA、TikTok native style、使用到的 viral pattern、迁移结果和剧情压过产品风险。
- 动作：查看、复制、标记通过、标记重做、重新导出任务包。
- 约束：只有素材充足且批次规划存在时才读取 `video_04/05`。
