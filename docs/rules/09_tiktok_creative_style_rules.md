# TikTok 原生短视频创意结构规则

> 本文档来源于 `00_design.md` 第 17 章，仅用于阅读，不是权威源；如与 `00_design.md` 冲突，以 `00_design.md` 为准。

## 基础结构与时间段

每条正式视频遵守 `Hook → Problem → Product → Proof → CTA`，不同分型可调整表达顺序，但不能缺少 Hook、Proof 或 CTA。

- 0—3 秒：用具体痛点、强结果、反差、问题、人群点名、失败、异常或强视觉变化截停用户。必须写明 `first_3_seconds_text`、`first_3_seconds_visual`、`scroll_stop_reason` 和 `product_relevance`。
- 3—8 秒：放大痛点或建立冲突，进入真实场景，不能只重复钩子。
- 8—18 秒：产品清晰进入并用动作、效果或细节证明核心卖点，禁止只说不演示。
- 18—25 秒：用前后对比、细节、结果、反应、参数/材料或真实场景补强信任；缺参数时用占位符。
- 结尾 2—3 秒：给出与痛点和结果相关的清晰 CTA。

## hook_type 类型库

每条视频至少选择一个：`pain_point_direct`（痛点直击）、`result_first`（结果先行）、`failed_method`（失败对比）、`counterintuitive`（反常识）、`audience_callout`（人群点名）、`scene_entry`（场景代入）、`suspense_reverse`（悬念反转）、`test_challenge`（测评挑战）、`price_value`（价格/价值）。

前 3 秒禁止品牌自我介绍、慢速外观展示、“今天介绍一款产品”、空泛形容词、公司/工厂/参数铺垫或把卖点藏到 5 秒以后。

## TikTok 原生风格判断

通过倾向：直接进入冲突/结果、朋友推荐语气、真实使用感、短而口语化字幕、产品自然进入、快而不乱、生活化但可识别、声音服务节奏、自然 CTA。

风险倾向：Logo 长开场、慢速旋转、大段参数、无场景棚拍、说明书字幕、培训课口播、只有品牌口号。出现风险时，应强化前 3 秒、减少品牌铺垫、增加场景和结果/对比、缩短字幕并补强 CTA。

## 字幕与文字叠加

前 3 秒必须有字幕或强画面信息；每屏一个意思，短句、口语化，卖点在动作发生时出现，不堆满屏幕、不使用复杂书面语、不遮挡产品。每条视频必须生成 `text_overlay_plan`，覆盖痛点、动作、证明、对比、风险或 CTA 中的适用项。

## 声音、音乐与口播

可使用原生口播、旁白、环境音、动作音效、节奏音乐、转场音效或无口播字幕流。声音服务节奏，口播不机械重复字幕，动作音效可强化证明感，音乐不盖过核心信息；无口播时必须靠画面与字幕讲清。每条视频必须生成 `sound_or_voiceover_plan`。

## 中段留人

每条视频必须生成 `retention_design`。可用每 2—4 秒视觉变化、失败方法转产品方法、脏乱转清洁、局部转结果、结果悬念、逐步证明等机制；不能把中段写成静态产品说明。

## 结尾 CTA

CTA 必须关联前述痛点和结果，清楚指向下一步，不编造优惠、库存、销量或效果。类型为 `product_card_click`、`comment_question`、`save_for_later`、`compare_before_buy`、`limited_offer`、`scenario_action`。

## 视频分型对应结构

- 生活场景型：场景痛点 → 使用麻烦 → 产品解决 → 结果与场景 CTA。
- 卖点讲解型：强卖点/痛点 → 核心功能 → 连续证明 2—3 个卖点 → 人群与 CTA。
- 开箱直拍型：开箱结果/包装亮点 → 包装配件 → 细节与使用价值 → 适合人群与 CTA；效果型商品不得以开箱为主力。
- 买家卖家秀型：真人效果/差异 → 使用前痛点 → 过程步骤 → 效果、感受与 CTA。
- 测评对比型：挑战/悬念 → 对象和标准 → 对比过程 → 结果、购买建议与 CTA。

## 类目钩子偏好

- 车载用品：优先痛点直击、失败对比、场景代入、测评挑战、反常识；画面优先车内缝隙、脚垫、杯架、后备箱和前后对比。
- 美妆个护：优先结果先行、前后对比、人群点名、步骤教程、场景代入；画面优先真人前后、局部、步骤、节时和安全细节。
- 家居/小家电：优先痛点、结果、生活场景、反常识、省时间；画面优先真实家庭场景、前后变化、功能动作和细节证明。
- 宠物用品：优先真实反应、场景、安全材质、耐用测试、情绪价值；画面优先互动、使用反应、安全、耐用和清洁便利。

类目偏好中的非正式类型名称与正式 hook 枚举的映射尚未定义，保留为设计问题，不在阅读版裁决。

## Codex 输出强制字段

`hook`（含四个前 3 秒字段及产品关联）、`script_structure`、`retention_design`、`key_message`、`proof_point`、`cta`、`tiktok_native_style`、`text_overlay_plan`、`sound_or_voiceover_plan`。缺少 hook、proof 或 CTA 不能视为完整视频资产。

## 禁止事项

不得写成传统广告片、占用前 3 秒介绍品牌、所有视频使用同一种钩子、只有脚本没有证明画面、只有外观没有场景、只有口号没有行动、为剧情牺牲产品清晰度、为电影感牺牲原生感，或编造优惠、库存、销量、参数和认证。

## 通过条件

已选择 hook 类型并完成前 3 秒四项说明；完成各时间段结构、key message、proof、retention、字幕、声音和 CTA；通过原生风格检查；未违反产品清晰度和禁止编造规则。字段不完整时只允许导出标明“TikTok 创意结构不完整”的草案，不能生成正式完整资产。
